**3. UserTable (`components/admin/users/user-table.tsx`):**
- Add `initialUserId?: string` to the component's props
- Add a `useEffect` that fires once on mount: if `initialUserId` is set and a user with that ID exists in the users array, auto-open the edit dialog for that user:
```tsx
useEffect(() => {
  if (initialUserId) {
    const targetUser = users.find((u) => u.id === initialUserId);
    if (targetUser) {
      // Set the edit dialog state to open with this user
      handleEdit(targetUser);
    }
  }
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, [initialUserId]);
```
- This assumes `handleEdit` sets state to open the `UserFormDialog` in edit mode. Check the existing implementation — if UserTable uses state like `editingUser` and `dialogOpen`, set those.

**4. URL update on edit (optional but recommended):**
When editing a user from the table (clicking edit button), update the URL with `userid` param using `window.history.replaceState` or `useQueryState` from nuqs:
```tsx
// When opening edit dialog:
const url = new URL(window.location.href);
url.searchParams.set('userid', user.id);
window.history.replaceState({}, '', url.toString());

// When closing edit dialog:
const url = new URL(window.location.href);
url.searchParams.delete('userid');
window.history.replaceState({}, '', url.toString());
```
This makes the current URL copyable/shareable while editing.
