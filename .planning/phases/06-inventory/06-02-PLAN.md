**Asset list page (app/(dashboard)/inventory/page.tsx):**

Follow `app/(dashboard)/requests/page.tsx` pattern exactly:
- Server component that fetches assets from Supabase
- Query: `supabase.from('inventory_items').select('*, category:categories(name), location:locations(name)').eq('company_id', profile.company_id).is('deleted_at', null).order('created_at', { ascending: false })`
- All roles can view (INVENTORY_VIEW_ALL permission — already assigned in permissions.ts)
- For each asset, also check if there's a pending movement to show "In Transit" indicator: do a separate query for pending movements: `supabase.from('inventory_movements').select('id, item_id, to_location_id, to_location:locations!to_location_id(name), receiver_id').in('item_id', assetIds).eq('status', 'pending').is('deleted_at', null)`
- Build a map of assetId -> pendingTransfer to pass to the table
- Also fetch categories and locations for filter dropdowns
- Pass data to `<AssetTable>` client component

Follow `components/requests/request-submit-form.tsx` pattern with two-step create (entity first, files second):

Form fields:
- Name (Input, required, maxLength=100)
- Category (Combobox, required — filtered to asset-type categories only)
- Location (Combobox, required)
- Brand (Input, optional, maxLength=100)
- Model (Input, optional, maxLength=100)
- Serial Number (Input, optional, maxLength=100)
- Description (Textarea, optional, maxLength=200)
- Acquisition Date (Input type="date", optional)
- Warranty Expiry (Input type="date", optional)
- Condition Photos (AssetPhotoUpload, required=true, maxPhotos=5)
- Invoice Files (separate file input area, optional, max 5 files, accept image/*,application/pdf, max 10MB each)

Layout: use a Card with form sections. Arrange required fields at top, optional fields below. Per CONTEXT "Claude's Discretion" for form layout — group logically:
- Section 1: Basic Info (name, category, location)
- Section 2: Identification (brand, model, serial number)
- Section 3: Dates (acquisition date, warranty expiry)
- Section 4: Description
- Section 5: Condition Photos (required)
- Section 6: Invoice Files (optional)

Use react-hook-form with zodResolver(assetCreateSchema).

On submit:
1. Call `createAsset` server action with form data
2. If success and assetId returned:

   b. Upload invoice files (if any): POST to `/api/uploads/asset-invoices` with FormData containing `asset_id` and invoice files
3. Show success feedback via InlineFeedback (persistent, not auto-dismiss)
4. Redirect to `/inventory/${assetId}` after successful upload

Error handling:
- If createAsset fails, show error via InlineFeedback
- If photo upload fails after asset creation, show error but don't lose the asset (it was already created)
- Disable submit button during processing, show loading state

Use InlineFeedback component for all feedback (per CLAUDE.md — never auto-dismiss).
