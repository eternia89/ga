**Detail page (app/(dashboard)/inventory/[id]/page.tsx):**

Follow `app/(dashboard)/requests/[id]/page.tsx` pattern:
- Server component
- Breadcrumb: Inventory > {display_id}
- Fetch asset with relations: `inventory_items` joined with category (name), location (name)
- Fetch condition photos via `getAssetPhotos` action (or direct query with signed URLs)
- Fetch invoices via `getAssetInvoices` action (or direct query with signed URLs)
- Fetch pending transfer (if any): `inventory_movements` WHERE item_id=id AND status='pending' AND deleted_at IS NULL, joined with to_location (name), receiver (full_name), initiator (full_name)
- Fetch audit_logs: `audit_logs` WHERE table_name='inventory_items' AND record_id=id, ordered by performed_at ASC
- Fetch movements: `inventory_movements` WHERE item_id=id AND deleted_at IS NULL, ordered by created_at ASC, joined with from_location (name), to_location (name), initiator (full_name), receiver (full_name)
- Fetch movement photos: `media_attachments` WHERE entity_type IN ('asset_transfer_send', 'asset_transfer_receive', 'asset_transfer_reject') AND entity_id IN (movementIds)
- Fetch status change photos: `media_attachments` WHERE entity_type IN ('asset_creation', 'asset_status_change') AND entity_id=id
- Generate signed URLs for all photos
- Fetch categories (asset type) and locations for edit form dropdowns
- Fetch GA Staff/Lead users for transfer receiver selection
- Pass all data to `<AssetDetailClient>`

**Layout:** Two-column grid per request detail pattern:
- `grid grid-cols-[1fr_380px] max-lg:grid-cols-1 gap-6`
- Left: Info panel + action buttons
- Right: Timeline

**Timeline (components/assets/asset-timeline.tsx):**

Follow `components/requests/request-timeline.tsx` pattern but with MORE event types. The asset timeline is a unified chronological view combining audit_logs and inventory_movements.

Event types to display:
1. **Created** — from audit_logs (action='INSERT' on inventory_items). Show "Asset created" with creation date. Include creation condition photo thumbnails.
2. **Field Updated** — from audit_logs (action='UPDATE' on inventory_items, excluding status-only changes). Show changed fields.
3. **Status Changed** — from audit_logs (action='UPDATE' where old_data.status != new_data.status). Show "Status: {old} -> {new}" with status labels. Include status change condition photo thumbnails.
4. **Transfer Initiated** — from inventory_movements record creation. Show "Transfer initiated: {from_location} -> {to_location}, Receiver: {receiver_name}". Include sender condition photo thumbnails.
5. **Transfer Accepted** — from inventory_movements (status='accepted'). Show "Transfer accepted by {receiver_name}". Include receiver condition photo thumbnails.
6. **Transfer Rejected** — from inventory_movements (status='rejected'). Show "Transfer rejected by {receiver_name}: {rejection_reason}". Include rejection photo thumbnails.
7. **Transfer Cancelled** — from inventory_movements (status='cancelled'). Show "Transfer cancelled by {initiator_name}".

Implementation:
- Receive: auditLogs, movements, photos (creation/status), transferPhotos (send/receive/reject) as props
- Merge all events into a single chronological array sorted by timestamp (performed_at for audit_logs, created_at/received_at/rejected_at/cancelled_at for movements)
- Each timeline entry has: timestamp, type, icon, title, description, optional photos
- Icons (from lucide-react): Plus for created, Pencil for field update, RefreshCw for status change, Truck for transfer events, Check for accepted, X for rejected, Ban for cancelled
- Photo thumbnails: small grid (max 3 visible + "+N more" indicator), clickable to open PhotoLightbox
- Timestamps formatted as dd-MM-yyyy, HH:mm:ss per CLAUDE.md
- Wrap in ScrollArea for the right column
- Consistent visual style with request timeline (vertical line, icon dots, etc.)
