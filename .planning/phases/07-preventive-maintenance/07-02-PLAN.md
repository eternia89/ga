**Template builder (components/maintenance/template-builder.tsx):**
This is the core UI component. It must be a client component ('use client').

Structure:
- Props: `{ items: ChecklistItem[], onItemsChange: (items: ChecklistItem[]) => void }`
- Uses `DndContext` with `closestCenter` collision detection and `PointerSensor` + `KeyboardSensor` (with `sortableKeyboardCoordinates`)
- Contains `SortableContext` with `verticalListSortingStrategy`
- `handleDragEnd`: uses `arrayMove` from @dnd-kit/sortable, then updates sort_order on each item
- **Type-specific add buttons row** (per CONTEXT.md locked decision): 6 buttons labeled "+ Checkbox", "+ Pass/Fail", "+ Numeric", "+ Text", "+ Photo", "+ Dropdown". Each generates a new ChecklistItem with `crypto.randomUUID()` for id, the selected type, empty label, and sort_order = items.length. For dropdown type, initialize with `options: ['']`.
- Renders each item via `<TemplateBuilderItem>` inside the SortableContext
- At the bottom: shows count of items (e.g., "3 items") and a message if empty ("Add at least one checklist item")

**Styling notes (desktop-first per CLAUDE.md):**
- Add buttons row: flex wrap with gap-2, each button uses a small outlined/ghost variant style
- Item list: vertical stack with gap-2, each item has a card-like border with rounded corners
- Use Tailwind max-* breakpoints if any responsive adjustments needed

**Sortable item (components/maintenance/template-builder-item.tsx):**
Each item is a sortable row using `useSortable` hook from @dnd-kit/sortable.

Structure:
- Props: `{ item: ChecklistItem, onUpdate: (item: ChecklistItem) => void, onDelete: (id: string) => void }`
- Uses `useSortable({ id: item.id })` to get transform, transition, setNodeRef, listeners, attributes
- Applies `CSS.Transform.toString(transform)` from @dnd-kit/utilities for smooth drag styling
- Layout: horizontal row with:
  - Drag handle (grip icon, uses `{...listeners}` and `{...attributes}`)
  - Type badge (using CHECKLIST_TYPES label and a small colored badge)
  - Label input: `<Input>` with placeholder "Item label..." maxLength={200}, value bound to item.label, onChange updates via onUpdate
  - **Type-specific configuration:**
    - `checkbox`: no extra config
    - `pass_fail`: no extra config
    - `numeric`: optional unit input `<Input>` with placeholder "Unit (e.g., PSI)" maxLength={20}
    - `text`: no extra config
    - `photo`: no extra config
    - `dropdown`: chips UX for options. Show existing options as removable chips (X button). Add new option via small input + "Add" button. Min 1 option, max 20 per RESEARCH.md schema. Each option max 100 chars.
  - Delete button (X icon, calls onDelete)
- Disabled state during drag (opacity reduced, pointer-events none on inputs)

**Template columns (components/maintenance/template-columns.tsx):**
TanStack Table column definitions for the template list. Columns per CONTEXT.md:
1. Name — text, clickable link to detail page `/maintenance/templates/{id}`
2. Category — joined category name
3. Items — count of checklist items (computed from checklist array length)
4. Created — formatted as dd-MM-yyyy per CLAUDE.md
5. Status — "Active" green badge or "Inactive" gray badge based on is_active
6. Actions — deactivate/reactivate button (only for ga_lead/admin)

**Template list (components/maintenance/template-list.tsx):**
Client component wrapping the reusable DataTable with template columns. Props: `{ templates: MaintenanceTemplate[], userRole: Role }`.
