**Schedule form (components/maintenance/schedule-form.tsx):**
Client component with react-hook-form + zodResolver(scheduleCreateSchema).

Form fields per CONTEXT.md locked decisions:
1. **Template** — Combobox (searchable dropdown). Initially loads all active templates. When a template is selected, filter the asset dropdown to only show assets matching that template's category. If a category is already implied (e.g., opened from template detail page), pre-select the template and lock it.
2. **Asset** — Combobox (searchable dropdown). Shows assets filtered by the selected template's category. If opened from asset detail page, pre-select the asset and lock it.
3. **Interval (days)** — numeric Input, min 1, max 365. maxLength not applicable (number input).
4. **Interval Type** — toggle/switch or radio group with two options: Fixed, Floating (default: Floating per CONTEXT.md). Include inline help text per CONTEXT.md locked decision: "Fixed = every N days from start date. Floating = N days after last completion."
5. **Start Date** — optional date picker. If omitted, defaults to now (handled server-side).

Props: `{ templates: MaintenanceTemplate[], assets: AssetListItem[], defaultTemplateId?: string, defaultAssetId?: string, mode: 'create' | 'edit', schedule?: MaintenanceSchedule }`

**Category filtering logic:**
- When template is selected, derive its category_id, filter assets to those with matching category_id
- When asset is selected first (from asset detail entry point), derive its category_id, filter templates to those with matching category_id
- This is a bidirectional filter. Use controlled state to track which was selected first.

Submit calls createSchedule or updateSchedule server action. On success: redirect to /maintenance with InlineFeedback. On error: show InlineFeedback with error.

**Status badge (components/maintenance/schedule-status-badge.tsx):**
Simple presentational component. Props: `{ schedule: { is_active: boolean, is_paused: boolean, paused_reason: string | null } }`.
- Calls `getScheduleDisplayStatus(schedule)` from lib/constants/schedule-status.ts
- Renders a badge with label from SCHEDULE_STATUS_LABELS and color from SCHEDULE_STATUS_COLORS
- Active: green, Paused (auto): amber, Paused (manual): yellow, Deactivated: gray

**Schedule columns (components/maintenance/schedule-columns.tsx):**
TanStack Table column definitions per CONTEXT.md locked decisions:
1. Template Name — text, from joined template.name
2. Asset Name — text, from joined asset.name. Optionally show asset.display_id as secondary text
3. Interval — number with "days" suffix (e.g., "30 days")
4. Type — "Fixed" or "Floating" badge
5. Status — ScheduleStatusBadge component
6. Next Due — formatted as dd-MM-yyyy per CLAUDE.md. Show "N/A" if paused or deactivated. Show red text if overdue (next_due_at < now).
7. Last Completed — formatted as dd-MM-yyyy, or "-" if never completed
8. Actions — deactivate/activate/delete buttons (ga_lead/admin only)

**Schedule list (components/maintenance/schedule-list.tsx):**
Client component wrapping DataTable with schedule columns. Props: `{ schedules: MaintenanceSchedule[], userRole: Role }`.
