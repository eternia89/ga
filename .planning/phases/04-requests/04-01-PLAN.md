```sql
ALTER TABLE public.requests ADD CONSTRAINT requests_status_check
  CHECK (status IN ('submitted', 'triaged', 'in_progress', 'pending_approval', 'approved', 'rejected', 'completed', 'accepted', 'closed', 'cancelled'));
```

2. **Create `generate_request_display_id` function** — uses 2-digit year, fetches company code, year-scoped counter:
```sql
CREATE OR REPLACE FUNCTION public.generate_request_display_id(p_company_id uuid)
RETURNS text
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_next_value bigint;
  v_year_key text;
  v_company_code text;
BEGIN
  v_year_key := TO_CHAR(NOW(), 'YY');

  SELECT code INTO v_company_code
  FROM public.companies
  WHERE id = p_company_id AND deleted_at IS NULL;

  UPDATE public.id_counters
  SET current_value = current_value + 1, updated_at = now()
  WHERE company_id = p_company_id AND entity_type = 'request_' || v_year_key
  RETURNING current_value INTO v_next_value;

  IF NOT FOUND THEN
    INSERT INTO public.id_counters (id, company_id, entity_type, prefix, current_value, reset_period)
    VALUES (gen_random_uuid(), p_company_id, 'request_' || v_year_key, COALESCE(v_company_code, 'REQ'), 1, 'yearly')
    RETURNING current_value INTO v_next_value;
  END IF;

  RETURN COALESCE(v_company_code, 'REQ') || '-' || v_year_key || '-' || LPAD(v_next_value::text, 4, '0');
END;
$$;
```
Mark as `SECURITY DEFINER` with `SET search_path = public` so it can read companies and update id_counters regardless of RLS.

**Step 1: Create all request server actions** at `app/actions/request-actions.ts`:

Use `authActionClient` (NOT adminActionClient) for ALL actions. Check role inside actions that need elevated permissions.

**`createRequest`** action:
- Schema: `requestSubmitSchema`
- Guard: Check `profile.division_id` is not null — throw "Your account has no division assigned. Contact your administrator." if missing
- Call `supabase.rpc('generate_request_display_id', { p_company_id: profile.company_id })` for atomic display_id
- Auto-generate title: `parsedInput.description.replace(/\s+/g, ' ').trim().slice(0, 100)`
- Insert into `requests` table: company_id, division_id (from profile — auto-filled per REQ-REQ-002), location_id, requester_id (profile.id), display_id, title, description, status: 'submitted'
- Use `.select('id, display_id').single()` to return the new request's ID
- `revalidatePath('/requests')`
- Return `{ success: true, requestId: data.id, displayId: data.display_id }`

**`updateRequest`** action:
- Schema: `z.object({ id: z.string().uuid(), data: requestEditSchema })`
- Guard: Only requester can edit, only while status = 'submitted'
- Query request first: `.eq('id', parsedInput.id).eq('requester_id', profile.id).eq('status', 'submitted').single()`
- If not found, throw "Cannot edit — request is not in New status or you are not the requester"
- Update description, location_id. Regenerate title from new description.
- `revalidatePath('/requests')`

**`triageRequest`** action:
- Schema: `z.object({ id: z.string().uuid(), data: triageSchema })`
- Role check: `if (!['ga_lead', 'admin'].includes(profile.role)) throw new Error('Triage access required')`
- Fetch request, verify `status === 'submitted'` — throw "Request can only be triaged when in New status" otherwise
- Update: category_id, priority, assigned_to, status = 'triaged'
- `revalidatePath('/requests')`

**`cancelRequest`** action:
- Schema: `z.object({ id: z.string().uuid() })`
- Update with guards: `.eq('id', parsedInput.id).eq('requester_id', profile.id).eq('status', 'submitted')`
- Set status = 'cancelled'
- Check affected rows (use `.select()` to verify update happened) — if no rows, throw "Cannot cancel — request is not in New status or you are not the requester"
- `revalidatePath('/requests')`

**`rejectRequest`** action:
- Schema: `z.object({ id: z.string().uuid(), data: rejectSchema })`
- Role check: ga_lead or admin only
- Verify status is 'submitted' or 'triaged' (can reject from either per CONTEXT)
- Update: status = 'rejected', rejection_reason, rejected_at = new Date().toISOString(), rejected_by = profile.id
- `revalidatePath('/requests')`

**`getRequestPhotos`** action (non-mutating, for signed URLs):
- Schema: `z.object({ requestId: z.string().uuid() })`
- Query `media_attachments` where entity_type = 'request' AND entity_id = requestId AND deleted_at IS NULL
- Generate signed URLs (6 hour expiry) using `supabase.storage.from('request-photos').createSignedUrls(paths, 21600)`
- Return array of `{ id, fileName, url, mimeType }`

**Step 2: Create photo upload API route** at `app/api/uploads/request-photos/route.ts`:

Use an API route (not server action) to avoid body size limits. Export a `POST` handler.

- Parse the multipart form data using `request.formData()`
- Extract `request_id` (string field) and files from the form
- Validate auth: create supabase server client, call `supabase.auth.getUser()`, fetch profile
- Validate request_id belongs to a request the user can upload to:
  - Query the request: must exist, status = 'submitted', requester_id = user.id (only requester can upload photos)
- Validate files: max 3 files, each max 5MB, only image/jpeg, image/png, image/webp
- Check existing attachment count: `media_attachments` where entity_type = 'request' AND entity_id = request_id — total (existing + new) must not exceed 3
- Upload each file to Supabase Storage using admin client (service_role) at path `{company_id}/{request_id}/{crypto.randomUUID()}-{sanitized_filename}`
- Insert `media_attachments` records for each uploaded file: company_id, entity_type = 'request', entity_id = request_id, file_name, file_path (from upload), file_size, mime_type, sort_order (based on existing count + index), uploaded_by = user.id
- Return JSON: `{ success: true, count: uploaded_count }`
- On error: return appropriate status code (401, 400, 500) with error message
