**A. Create `app/(dashboard)/approvals/page.tsx`** — server component:
- Check user role: only finance_approver and admin can access. If unauthorized, redirect to `/`.
- Fetch jobs with status `pending_approval` (pending tab data):
  ```
  supabase.from('jobs').select('*, location:locations(name), category:categories(name), pic:user_profiles!assigned_to(full_name), created_by_user:user_profiles!created_by(full_name), job_requests(request:requests(display_id, title, requester:user_profiles!requester_id(full_name)))')
    .eq('status', 'pending_approval').is('deleted_at', null)
    .order('approval_submitted_at', { ascending: true })
  ```
- Fetch jobs with status NOT `pending_approval` but where `approved_at IS NOT NULL` OR `approval_rejected_at IS NOT NULL` (history tab data):
  ```
  supabase.from('jobs').select('*, pic:user_profiles!assigned_to(full_name), approved_by_user:user_profiles!approved_by(full_name), rejected_by_user:user_profiles!approval_rejected_by(full_name)')
    .or('approved_at.not.is.null,approval_rejected_at.not.is.null')
    .is('deleted_at', null)
    .order('updated_at', { ascending: false })
    .limit(50)
  ```
- Render breadcrumb: Approvals.
- Pass both datasets to `<ApprovalQueue />`.

**B. Create `components/approvals/approval-queue.tsx`** — client component:
- Two tabs using shadcn Tabs component: "Pending" (default) and "History".
- **Pending tab**:
  - Table/card list showing: Job display_id, title, estimated cost (IDR formatted prominently), PIC name, linked request(s) display_ids, submission date (dd-MM-yyyy).
  - If no pending items: empty state message "No jobs awaiting approval".
  - Row click navigates to `/jobs/[id]` — per CONTEXT.md locked decision, approve/reject happens on the job detail page, NOT inline here.
  - Sort by submission date ascending (oldest first = most urgent).

- **History tab**:
  - Table showing: Job display_id, title, estimated cost (IDR), decision ("Approved" or "Rejected" badge), decided by, decision date (dd-MM-yyyy).
  - For rejected items: show rejection reason in a secondary line or tooltip.
  - Row click navigates to `/jobs/[id]`.
  - Sort by decision date descending (most recent first).

- Use the same table styling as other data tables in the app (consistent with TanStack Table look). Can use a simpler table here (no TanStack needed since no client-side filtering/sorting — server already sorted).
