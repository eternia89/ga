  <action>
**A. Create `app/(dashboard)/jobs/new/page.tsx`** — server component:
- Fetch locations (filtered by company, active only), categories (type='request', active only), GA staff + GA lead users (for PIC assignment), and eligible requests (status 'triaged' — these can be linked to jobs).
- Check for optional `?request_id=` query param. If present, pre-fetch that request's data for form prefill.
- Render breadcrumb: Jobs > New Job.
- Pass all reference data + optional prefill to `<JobForm />`.

**B. Create `components/jobs/job-form.tsx`** — client component:
- Props: locations, categories, users (for PIC), eligibleRequests, prefillRequest (optional), mode ('create' | 'edit').
- Use react-hook-form with zodResolver(createJobSchema).
- Fields:
  1. **Title** — Input maxLength={150}. Pre-filled from request title if creating from request.
  2. **Description** — Textarea maxLength={1000}. Pre-filled from request description.
  3. **Location** — Combobox (reuse existing component). Pre-filled from request location.
  4. **Category** — Combobox. Pre-filled from request category.
  5. **Priority** — Select (Low/Medium/High/Urgent). Pre-filled: if creating from request, inherit request priority; if multiple requests linked, auto-set to highest priority.
  6. **PIC (Person in Charge)** — Combobox. Optional at creation. Only show GA Staff and GA Lead users.
  7. **Estimated Cost** — Input type="number" with "Rp" prefix label. Optional. Format display with IDR formatter.
  8. **Linked Requests** — Multi-select Combobox or a list with add/remove:
     - Show a Combobox that searches eligible requests by display_id or title.
     - Selected requests appear as chips/tags below the dropdown with remove buttons.
     - Requests already linked to other jobs should be shown in the dropdown but marked with "(linked to JOB-XX-XXXX)" — they CAN still be selected (per CONTEXT.md: shown but marked).
     - If creating from a request (prefill), that request is pre-added to the linked list.
     - When requests are added/removed, auto-recalculate priority to the highest among linked requests (only override if linked requests exist and current priority is lower).

- On submit: call `createJob` server action. If PIC is assigned, the server action will handle setting status to 'assigned'. Redirect to `/jobs/[id]` on success using router.push.
- Use InlineFeedback for error display (persistent, user-dismissable).
- Desktop-first layout per CLAUDE.md: form fields in a reasonable single-column layout for the form area.
