  <name>Task 1: Fix approval queue FK join failure</name>
  <files>app/(dashboard)/approvals/page.tsx</files>
  <action>
The approval queue Supabase query uses FK join hints `user_profiles!approved_by` and `user_profiles!approval_rejected_by` which PostgREST cannot resolve because the jobs table has 6 FK constraints to user_profiles. The query silently returns null (error not captured).

Fix by:
1. Remove the broken FK joins from the main query select string. Keep ONLY `pic:user_profiles!assigned_to(full_name)` which works (only 2 FKs for jobs list page work). Remove `approved_by_user:user_profiles!approved_by(full_name)` and `rejected_by_user:user_profiles!approval_rejected_by(full_name)`.
2. Add `approved_by` and `approval_rejected_by` as plain UUID columns in the select (they are already columns on the jobs table).
3. After the main query, batch-fetch approver/rejecter names via a separate query (same pattern as the existing `completionActorMap` fetch below). Collect all unique `approved_by` and `approval_rejected_by` UUIDs, fetch their `full_name` from user_profiles, build a lookup map.
4. In the `for (const job of rawJobs)` loop, replace `approvedByUserNorm` and `rejectedByUserNorm` with lookups from the new map instead of FK join data. Build `{ full_name: string }` objects from the map to match the existing `ApprovalJob` type shape.
5. Add proper `{ data, error }` destructuring on the main query. If error, log it with `console.error` and default to empty array.

The `completionActorMap` pattern already exists in this file (lines 74-94) for completion approver names. Follow the exact same pattern for budget approver/rejecter names — combine all actor IDs into a single batch query for efficiency.
