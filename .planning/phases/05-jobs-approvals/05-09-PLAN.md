    **Refactor approval queue (UAT test 18):**

    The current `approval-queue.tsx` uses `Tabs` (Pending/History) with two separate simple HTML tables. Refactor to a single data table view:

    **Server component (`app/(dashboard)/approvals/page.tsx`):**
    1. Keep the same data fetching but unify: fetch ALL jobs that either have `status = 'pending_approval'` OR have `approved_at IS NOT NULL` OR have `approval_rejected_at IS NOT NULL`
    2. Pass the unified list to `ApprovalQueue` as a single `jobs` prop (combine pending + history into one array, with a computed `decision` field: 'pending' | 'approved' | 'rejected')

    **Client component (`components/approvals/approval-queue.tsx`):**
    1. Remove the `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent` imports and structure
    2. Replace with a single table view:
       - A `Checkbox` at the top with label "Show approved history" (default: unchecked)
       - When unchecked: show only `decision === 'pending'` items
       - When checked: show all items (pending + approved + rejected)
    3. Table columns:
       - Job ID (font-mono)
       - Title (with rejection reason below if rejected)
       - Estimated Cost (IDR formatted, prominent)
       - PIC name
       - Status: Badge showing "Pending" (yellow), "Approved" (green), or "Rejected" (red)
       - Submitted/Decided date
    4. Keep the row click navigation to `/jobs/[id]`
    5. Keep the empty state when no pending items (different message when filtered vs unfiltered)
    6. Sort: pending items first, then by date descending

    **Type updates:**
    Create a unified type:
    ```typescript
    type ApprovalJob = {
      id: string;
      display_id: string;
      title: string;
      estimated_cost: number | null;
      status: string;
      approval_submitted_at: string | null;
      approved_at: string | null;
      approval_rejected_at: string | null;
      approval_rejection_reason: string | null;
      created_at: string;
      pic?: { full_name: string } | null;
      approved_by_user?: { full_name: string } | null;
      rejected_by_user?: { full_name: string } | null;
      job_requests?: Array<{
        request: { display_id: string };
      }>;
      decision: 'pending' | 'approved' | 'rejected';
    };
    ```

    **Keep it simple:** Use a basic shadcn Table (not TanStack Table) since this is a compact, pre-sorted list. The only client-side interaction is the checkbox filter.
