    **Fix star rating auto-open (UAT test 14):**

    The issue: In `request-acceptance-dialog.tsx`, the `handleAccept` function calls `onSuccess()` then `onAccepted?.()`. The `onSuccess` triggers `router.refresh()` which causes a server re-render. The `onAccepted` callback sets `feedbackOpen: true` in the parent, but `router.refresh()` may cause the state to be lost or the timing to conflict.

    Fix approach â€” reorder the calls to ensure feedback dialog opens reliably:

    In `request-acceptance-dialog.tsx`, change the `handleAccept` function:
    ```typescript
    const handleAccept = async () => {
      setIsSubmitting(true);
      setErrorMessage(null);

      try {
        const result = await acceptRequest({ request_id: requestId });
        if (result?.serverError) {
          setErrorMessage(result.serverError);
          return;
        }
        onOpenChange(false);
        // Call onAccepted FIRST to set feedbackOpen state before refresh
        onAccepted?.();
        // Then refresh â€” React preserves client state across server refresh
        onSuccess();
      } catch (err) {
        setErrorMessage(err instanceof Error ? err.message : 'Failed to accept request');
      } finally {
        setIsSubmitting(false);
      }
    };
    ```

    The key change: call `onAccepted?.()` BEFORE `onSuccess()`. This ensures the `feedbackOpen` state is set to `true` before `router.refresh()` triggers a re-render. React preserves client state across `router.refresh()` calls, but timing matters â€” the state must be set first.

    Additionally, in `request-detail-actions.tsx`, wrap the `handleAccepted` callback with a small `setTimeout` as a safety net:
    ```typescript
    const handleAccepted = () => {
      // Use setTimeout to ensure state update happens after dialog close animation
      setTimeout(() => {
        setFeedbackOpen(true);
      }, 100);
    };
    ```
