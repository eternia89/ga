**A. Add actions to `app/actions/request-actions.ts`**:

1. `acceptRequest` — authActionClient. Schema: { request_id: uuid }. Verify the request is in `pending_acceptance` status. Verify the caller is the original requester (request.requester_id === profile.id) OR is admin. Set request status to `accepted`, accepted_at = now(). Revalidate `/requests` and `/requests/[id]`.

2. `rejectCompletedWork` — authActionClient. Schema: { request_id: uuid, reason: string 1-1000 }. Verify request is in `pending_acceptance`. Verify caller is requester or admin. Set request status to `in_progress`, store acceptance_rejected_reason = reason. Also update the linked job(s) (via job_requests join table): set each linked job's status back to `in_progress` and clear completed_at. This allows the PIC to rework. Revalidate.

3. `submitFeedback` — authActionClient. Schema: feedbackSchema from job-schema.ts (request_id, rating 1-5, comment optional max 200). Verify request is in `accepted` status. Verify caller is requester. Update request with feedback_rating and feedback_comment. Optionally set status to `closed` (the final terminal state, indicating feedback received). Revalidate.
