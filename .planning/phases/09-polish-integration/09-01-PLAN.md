**Migration (00008_job_status_changes.sql):**
Create the `job_status_changes` table for GPS-tracked job status transitions:
- Columns: id (uuid PK), job_id (uuid FK to jobs), company_id (uuid FK to companies), from_status (text NOT NULL), to_status (text NOT NULL), changed_by (uuid FK to user_profiles), latitude (double precision), longitude (double precision), gps_accuracy (double precision), created_at (timestamptz DEFAULT now())
- Indexes: idx_job_status_changes_job_id ON (job_id), idx_job_status_changes_company ON (company_id)
- RLS enabled with two policies:
  - `job_status_changes_select`: FOR SELECT TO authenticated USING (company_id = public.current_user_company_id())
  - `job_status_changes_insert`: FOR INSERT TO authenticated WITH CHECK (company_id = public.current_user_company_id())
- Follow the pattern from existing migrations (00007_requests_phase4.sql)

**useGeolocation hook (hooks/use-geolocation.ts):**
Create `hooks/` directory if it doesn't exist. Create a custom React hook:
- `'use client'` directive at top
- Export `useGeolocation` function returning `{ capturing: boolean, error: GpsError | null, capturePosition: () => Promise<GpsResult> }`
- `GpsResult` type: `{ latitude: number, longitude: number, accuracy: number }`
- `GpsError` type: `{ code: number, message: string }`
- `capturePosition` wraps `navigator.geolocation.getCurrentPosition` in a Promise
- Options: `enableHighAccuracy: true`, `timeout: 15000` (15 seconds), `maximumAge: 60000` (1 minute cache)
- Error messages: code 1 = "Location permission denied. Please allow location access to update job status.", code 2 = "Location unavailable. Please check your device settings.", code 3 = "Location request timed out. Please try again.", default for no geolocation = "Geolocation not supported by this browser"
- Use `useState` for capturing/error state, `useCallback` for capturePosition
