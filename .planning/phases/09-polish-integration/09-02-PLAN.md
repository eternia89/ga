**Audit trail page (page.tsx):**
Create the route at `app/(dashboard)/admin/audit-trail/page.tsx` as a server component:

1. Authentication and authorization: Use `createClient()` to get Supabase client, fetch current user and profile. Verify role is 'admin' or 'ga_lead' (per CONTEXT.md: "Accessible to Admin and GA Lead roles"). If unauthorized, redirect to `/unauthorized`.
2. Fetch audit logs with a default date range of last 30 days:
   ```
   const thirtyDaysAgo = subDays(new Date(), 30);
   const { data: logs } = await supabase
     .from('audit_logs')
     .select('*')
     .gte('performed_at', thirtyDaysAgo.toISOString())
     .order('performed_at', { ascending: false })
     .limit(1000);
   ```
3. Fetch all company users for the user filter dropdown:
   ```
   const { data: users } = await supabase
     .from('user_profiles')
     .select('id, full_name, email')
     .is('deleted_at', null)
     .order('full_name');
   ```
4. Resolve display IDs for entities that have them (requests, jobs, inventory_items). For each unique record_id in logs where table_name is requests/jobs/inventory_items, batch-query those tables to get display_id. Merge into log entries.
5. Resolve user names: Join performed_by UUIDs with the users list to add user_full_name/user_email to each log entry.
6. Render page header with title "Audit Trail" and description "View system activity logs".
7. Add breadcrumb: Dashboard > Settings > Audit Trail (breadcrumb implementation follows the existing pattern from request detail page)
8. Render the `AuditTrailTable` component with the enriched data and users list.
