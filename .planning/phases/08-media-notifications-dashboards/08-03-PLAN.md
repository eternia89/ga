  <action>
1. Create `lib/notifications/helpers.ts` (server-side only, no 'use client'):
   - Import `createAdminClient` from `@/lib/supabase/admin`
   - Define `NotifyParams` interface:
     ```
     companyId: string
     recipientIds: string[]     // users to notify
     actorId: string            // user who performed action (EXCLUDED from recipients per REQ-NOTIF-007)
     title: string              // max 150 chars
     body?: string              // max 200 chars
     type: string               // 'status_change' | 'assignment' | 'approval' | 'completion' | 'auto_accept_warning'
     entityType?: string        // 'request' | 'job' | 'inventory' | 'maintenance_schedule'
     entityId?: string          // UUID of the entity
     ```
   - Export `createNotifications(params: NotifyParams)`:
     - Filter out actorId from recipientIds (REQ-NOTIF-007: never notify actor about own action)
     - If filtered list is empty, return early
     - Build rows: one per recipient with `company_id, user_id, title, body, type, entity_type, entity_id`
     - Use `createAdminClient()` for INSERT (notifications table has no INSERT policy for regular users — INSERT via service_role only)
     - Wrap in try/catch — log errors but never throw (notification failure should never break the triggering action)

2. Create `lib/notifications/actions.ts` ('use server'):
   - Import `authActionClient` from `@/lib/safe-action`
   - Export `getUnreadCount` server action (no schema needed, uses ctx.profile):
     - Query `notifications` table: `select('id', { count: 'exact', head: true }).eq('user_id', profile.id).eq('is_read', false).is('deleted_at', null)`
     - Return `{ count: number }`
   - Export `getRecentNotifications` server action:
     - Query `notifications` table: `select('*').eq('user_id', profile.id).is('deleted_at', null).order('created_at', { ascending: false }).limit(10)`
     - Return `{ notifications: Notification[] }`
   - Export `markAsRead` server action (schema: `{ id: z.string().uuid() }`):
     - Update notification: `update({ is_read: true, read_at: new Date().toISOString() }).eq('id', id).eq('user_id', profile.id)`
     - Return `{ success: true }`
   - Export `markAllAsRead` server action (no schema):
     - Update all unread: `update({ is_read: true, read_at: new Date().toISOString() }).eq('user_id', profile.id).eq('is_read', false).is('deleted_at', null)`
     - Return `{ success: true }`
   - Define and export `Notification` type matching the DB schema

3. Create `lib/notifications/hooks.tsx` ('use client'):
   - Import `useState, useEffect, useCallback` from React
   - Import server actions from `./actions`
   - Export `useNotifications()` hook:
     - State: `unreadCount: number`, `notifications: Notification[]`, `isLoading: boolean`
     - `refresh` callback: calls `getUnreadCount()` and `getRecentNotifications()` in parallel, updates state
     - `useEffect` on mount: call `refresh()` immediately, then `setInterval(refresh, 30_000)` (30s polling per user decision)
     - CRITICAL: Return cleanup function from useEffect that calls `clearInterval` to prevent memory leaks
     - Use `useCallback` for refresh to maintain stable reference
     - Export `{ unreadCount, notifications, refresh, isLoading }`
  </action>
