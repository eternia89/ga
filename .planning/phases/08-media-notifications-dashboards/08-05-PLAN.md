  <action>
1. Create `lib/dashboard/queries.ts`:
   - Import types from supabase
   - Export `getDashboardKpis(supabase, companyId, dateRange: { from: string, to: string })`:
     - Run parallel count queries for:
       a. **Open Requests**: status IN ('submitted', 'triaged', 'in_progress'), deleted_at IS NULL
       b. **Untriaged Requests**: status = 'submitted', deleted_at IS NULL
       c. **Overdue Jobs**: status = 'in_progress', created_at < 7 days ago (proxy for "overdue" — since there's no explicit due date, use the age heuristic), deleted_at IS NULL
       d. **Completed This Period**: requests with status = 'accepted' or 'completed', updated_at within dateRange
       e. **Open Jobs**: status IN ('created', 'assigned', 'in_progress'), deleted_at IS NULL
     - For each KPI, also query the PREVIOUS period (same duration, shifted back) to calculate trend
     - Return array of KPI objects: `{ id, title, value, previousValue, href, trendIsGood }`
   - Export `calculateTrend(current: number, previous: number): { direction, percentage }`:
     - If previous is 0: direction = current > 0 ? 'up' : 'flat', percentage = 0
     - Otherwise: percentage = Math.abs(Math.round(((current - previous) / previous) * 100))
     - direction = change > 0 ? 'up' : change < 0 ? 'down' : 'flat'
   - NOTE: RLS will automatically scope queries to the user's company. The queries go through the authenticated Supabase client, so `deleted_at IS NULL` is handled by RLS SELECT policies.
