  <files>
    app/api/vision/describe/route.ts

  <action>
1. Create `app/api/vision/describe/route.ts`:
   - POST endpoint accepting `{ imageBase64: string, attachmentId: string }`
   - Auth check (getUser, verify profile exists and not deactivated)
   - Read `GOOGLE_VISION_API_KEY` from `process.env` (NO `NEXT_PUBLIC_` prefix — server-side only)
   - If API key is missing, return `{ description: null, error: 'Vision API not configured' }` with 200 status (graceful degradation — not a hard error)
   - Call Google Vision REST API: `POST https://vision.googleapis.com/v1/images:annotate?key=${apiKey}`
   - Request body: `{ requests: [{ image: { content: imageBase64 }, features: [{ type: 'LABEL_DETECTION', maxResults: 5 }] }] }`
   - Extract labels from `data.responses[0].labelAnnotations`, join descriptions with comma
   - Update `media_attachments.description` column for the given `attachmentId` using adminClient
   - Return `{ description }` on success
   - Wrap entire Vision API call in try/catch — if it fails, log error and return `{ description: null }` (never block the upload)

2. Update `app/api/uploads/entity-photos/route.ts` (from plan 08-01):
   - After each successful upload+insert, fire a non-blocking Vision API call:
     ```
     // Fire-and-forget: trigger Vision API description
     const fileBuffer = await file.arrayBuffer();
     const base64 = Buffer.from(fileBuffer).toString('base64');
     fetch(`${process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'}/api/vision/describe`, {
       method: 'POST',
       headers: { 'Content-Type': 'application/json', 'Cookie': request.headers.get('cookie') || '' },
       body: JSON.stringify({ imageBase64: base64, attachmentId: insertedId }),
     }).catch(() => {}); // Truly fire-and-forget
     ```
   - Alternative approach (simpler, preferred): Call the Vision API directly within the upload route instead of making an HTTP call to self. Read the file buffer, base64 encode, call the Vision REST API fetch directly, update media_attachments.description with adminClient. Wrap in try/catch, never block the upload response. Do NOT await — use `.then().catch()` pattern.
   - Return the insertedId (attachment row id) from the insert to use for the Vision update
  </action>
