<objective>
Add optimistic locking to updateAsset, updateJob, and updateRequest server actions to prevent concurrent edits from silently overwriting each other.

Purpose: Two users editing the same entity currently use last-write-wins. This adds version checking using the `updated_at` timestamp column (already present in all tables and already fetched by `*` select in detail pages) to detect and reject stale writes.

Output: Three server actions with optimistic locking + three form components passing `updated_at` to the actions.
</objective>

```typescript

if (parsedInput.updated_at && existing.updated_at !== parsedInput.updated_at) {
  throw new Error('This record was modified by another user. Please refresh the page and re-apply your changes.');
}
```

- **updateRequest**: After line 91 (`if (!existing)` block)
